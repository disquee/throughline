# VenueOS 2026.2

Target ship date: February 18, 2026.

## Smart Hold

The headline of this release. Venues can hold a space without confirming it, which removes the side spreadsheet that large venues used to track tentative bookings. See [Smart Hold](/features/smart-hold) for the full model.

Highlights:

- A new hold state with a configurable time limit, default 48 hours.
- Conflict detection that respects turnover and buffer time between events.
- A new API endpoint for creating holds. See the [Holds API](/api/holds).

## Reporting fix

The booking activity export now includes cancelled and expired holds, so funnel volume reported to Revenue Operations is no longer understated.

## For GTM teams

Until 2026.3, account execs check availability inside VenueOS rather than in Salesforce, and Customer Success should expect early questions about how to create a hold.
