# Configuring Booking Rules

This page predates Smart Hold and has not been updated for the 2026.2 changes. It still describes booking as a two-state DRAFT/CONFIRMED model and does not mention holds, TTL, or turnover buffers. Flagged here as a candidate for the staleness pass.

Venue admins manage booking behavior under **Admin > Booking Rules**. Current settings documented on this page:

- Default booking duration
- Cancellation window
- Required deposit percentage

Holds and turnover buffers are not yet documented here.
